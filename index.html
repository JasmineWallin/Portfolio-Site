New Accessions

New Accessions is one of the five programs that I have written at my current position as a Software Developer for Special Collections. It automates the process of adding new accessions into the Special Collections library database. There are two parts: new_accessions.py and update_repos.py.

Language: Python

Tech Stack/Toolkit: Pandas, ArhivesSnake API, JSON, Regex

Program Runthrogh

A spreadsheet is automatically filled with inputs from a google form, and downloaded as a CSV.


Running new_accessions.py program on the input csv.
A csv path is the second argument and input for this program, which outputs two logs (error and success) and 'posted_accessions.csv'. While going through each line of the input csv, the program first creates a new accession within the library database and then determines if a matching resource exists in the database, and writes this information in the output csv.

posted_accessions.csv after the run:


After senior-level Special Collections staff reviews the csv and logs, update_repos.py is run.
Update repos parses through the 'posted_accessions.csv' and either puts the new accession into the matching resource, or creates a new resource, depending on the 'Resource Found' column.


New Accessions are now in Library system and sorted in their respective repositories.
yay!

Feeble Finance


Feeble Finance is a GUI Application that I have created to manage personal finances. Users input their own monthly expenses and income and can see a breakdown of finances in a clear and concise manner through graphs, charts, and spreadsheets.

Language: Python

Tech Stack/Toolkit: Pandas, Pyqt5, Pymssql, Matplotlib, Microsoft SQL

Software Used: Docker Desktop, Azure Data Studio, VSCode

Program Runthrogh

Based on running on a A 2019 Ventura Intel Macbook.


Connect to or set up your own Microsoft SQL server.
Below is an example of how I connected to my own MSSQL server with a Docker container after initial setup and downloads. The programs used are Docker Desktop and Azure Data Studio.


Running the Feeble Finance GUI from main.py.
Upon running main.py after first downloading or cloning Feeble Finance and navigating to the Feeble Finance repository on your computer, this GUI should appear.


Adding New Financial Data
When starting the program for the first time with a new MS SQL server, the database is empty, so we'll need to fill it with some financial data. The 'My Data' tab has a button named 'Add More Data' where you can input new financial items.


Upon clicking the 'Add More Data' button, a new window with a form is shown in which you can input your financial data.


Clicking the 'Refresh Data' button will update the spreadsheet to show your new item.


Seeing Your Financial Graphs and Charts
After there is some financial data in the database, the program can start plotting your financial habits on the graphs and charts shown on the 'Graphs and Charts' tab.

Click the 'Refresh Data' button.


On the 'Graphs and Charts' tab, there is a graph on the timeline of your general spending habits as well as different categories of spending habits, a pie chart and bar graph of the categories of spending habits from the current month, and a list of your three greatest current expenses.
